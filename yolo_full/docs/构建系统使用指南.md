# 构建系统使用指南

## 📦 构建工具概述

本项目提供了多种构建工具，类似CMake的项目管理方式，支持跨平台使用：

### 可用的构建工具

| 工具 | 平台 | 说明 |
|------|------|------|
| `Makefile` | Linux/Mac | GNU Make构建脚本 |
| `build.bat` | Windows | 批处理构建脚本 |
| `build.sh` | Linux/Mac | Shell构建脚本 |
| `build.py` | 全平台 | Python构建脚本（推荐） |
| `CMakeLists.txt` | 全平台 | CMake构建配置 |

## 🚀 快速开始

### Windows用户

#### 方法1：使用批处理脚本（推荐）
```cmd
# 一键安装
build.bat all

# 启动应用
build.bat gui

# 查看帮助
build.bat help
```

#### 方法2：使用Python脚本
```cmd
# 一键安装
python build.py all

# 启动应用
python build.py gui

# 查看帮助
python build.py help
```

### Linux/Mac用户

#### 方法1：使用Makefile（推荐）
```bash
# 一键安装
make all

# 启动应用
make gui

# 查看帮助
make help
```

#### 方法2：使用Shell脚本
```bash
# 添加执行权限
chmod +x build.sh

# 一键安装
./build.sh all

# 启动应用
./build.sh gui

# 查看帮助
./build.sh help
```

#### 方法3：使用Python脚本
```bash
# 一键安装
python3 build.py all

# 启动应用
python3 build.py gui

# 查看帮助
python3 build.py help
```

### 使用CMake（高级用户）

```bash
# 创建构建目录
mkdir build && cd build

# 配置项目
cmake ..

# 执行构建任务
make init
make install
make gui

# 返回项目根目录
cd ..
```

## 🔧 自定义配置

### 修改Python解释器

#### Makefile
编辑 `Makefile`，修改：
```makefile
PYTHON = python3  # 改为你的Python路径
```

#### build.py
无需修改，自动使用当前Python解释器

#### CMakeLists.txt
```bash
cmake -DPYTHON_EXECUTABLE=/path/to/python ..
```

### 修改训练参数

编辑对应的构建脚本，修改命令行参数。例如在 `Makefile` 中：

```makefile
train-baseline: ## 训练基线模型
	@$(PYTHON) train/train_baseline.py \
		--data data/helmet.yaml \
		--weights models/weights/yolov5s.pt \
		--epochs 200 \  # 改为200轮
		--batch-size 32  # 改为32
```

## 🐛 常见问题

### Q1：命令找不到

**Windows:**
```cmd
# 如果build.bat不工作，尝试完整路径
D:\htools\order\1031\yolo\build.bat install
```

**Linux/Mac:**
```bash
# 添加执行权限
chmod +x build.sh

# 使用完整路径
/path/to/project/build.sh install
```

### Q2：Python找不到

确保Python在系统PATH中：
```bash
# 检查Python
python --version
python3 --version

# 或使用完整路径
/usr/bin/python3 build.py install
```

### Q3：Make命令不可用

**Windows:**
```cmd
# 使用build.bat或build.py代替
build.bat install
python build.py install
```

**Linux/Mac:**
```bash
# 安装make
# Ubuntu/Debian
sudo apt-get install build-essential

# macOS
xcode-select --install
```

### Q4：Git克隆失败

如果无法访问GitHub：
```bash
# 1. 使用代理
git config --global http.proxy http://proxy.example.com:8080

# 2. 或手动下载YOLOv5
# 访问 https://github.com/ultralytics/yolov5
# 下载ZIP并解压到 models/yolov5/
```

### Q5：权重下载失败

手动下载权重文件：
1. 访问：https://github.com/ultralytics/yolov5/releases
2. 下载 `yolov5s.pt`
3. 放到 `models/weights/` 目录

## 📖 进阶使用

### 添加自定义命令

#### 在Makefile中添加
```makefile
my-command: ## 我的自定义命令
	@echo "执行自定义命令"
	@$(PYTHON) my_script.py
```

#### 在build.py中添加
```python
def my_command(self):
    """我的自定义命令"""
    self.print_section("执行自定义命令...")
    return self.run_command([self.python, 'my_script.py'])

# 在commands字典中添加
commands = {
    # ...
    'my-command': builder.my_command,
}
```

### 集成CI/CD

在 `.github/workflows/build.yml` 中：
```yaml
name: Build and Test

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Install
        run: make install
      - name: Test
        run: make test
```

## 💡 最佳实践

1. **使用虚拟环境**
   - 始终在虚拟环境中工作
   - 避免污染系统Python环境

2. **定期清理**
   - 定期运行 `make clean`
   - 避免磁盘空间浪费

3. **版本控制**
   - 提交前运行 `make format lint test`
   - 确保代码质量

4. **文档更新**
   - 修改构建脚本后更新本文档
   - 保持文档与代码同步
